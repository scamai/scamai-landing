# Project Correction Rules

- [2026-06-12] verify-prod-build (async-race): Curling a `next start` build with `kill -9; sleep 1; next start` leaves the OLD process on the port (EADDRINUSE) so you test a STALE build — assert the port is free (`[ -z "$(lsof -ti:PORT)" ]`), use a FRESH port per rebuild, grep the server log for EADDRINUSE, and bake a unique marker into the page to confirm freshness BEFORE theorizing a root cause. This caused a multi-hour false "next-intl zh-CN/ar bug" hunt — the i18n was always fine.
- [2026-06-12] html-lang (settings-disconnect): Derive `<html lang>`/`dir` in the root layout from next-intl `getLocale()` — NOT from a cookie the middleware writes. The middleware read `X-NEXT-INTL-LOCALE` off the INCOMING request (absent there → always "en") and the cookie lands one request late, so lang/dir were stuck on the default and Arabic never got `dir="rtl"`.
- [2026-06-12] fix-small-mechanical (workflow): When a found problem is small, mechanical and decision-free (RTL/lang attribute, brand-name sweep, dead import, broken link) — fix and verify it directly. Don't ask "should I fix this?"; that burns a round-trip on a non-decision. Reserve questions for genuine forks.
